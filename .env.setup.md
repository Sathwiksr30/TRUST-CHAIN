# 🚀 TrustChain - Quick Setup Guide

This guide explains how to configure and run your TrustChain project with the centralized `.env` configuration file.

## 📋 Prerequisites

Before starting, ensure you have installed:
- **Node.js** (v16+) - [Download](https://nodejs.org/)
- **Python** (v3.8+) - [Download](https://www.python.org/)
- **Git** - [Download](https://git-scm.com/)

---

## ⚙️ Setup Instructions

### 1️⃣ Clone the Project
```bash
git clone <your-repo-url>
cd TRUST-CHAIN
```

### 2️⃣ Configure Environment Variables

**Option A - Quick Start (Default Configuration)**
```bash
# Copy the default config file to .env
cp .env.example .env
# That's it! The default values will work for local development
```

**Option B - Custom Configuration**
```bash
# Open the .env file and edit values as needed
# nano .env  (or use your preferred editor)
```

### 3️⃣ Install Dependencies

**Backend:**
```bash
cd trustchain-backend
npm install
cd ..
```

**Frontend:**
```bash
npm install
```

**ML Service (Python):**
```bash
cd ml
pip install -r requirements.txt
cd ..
```

**Blockchain (Hardhat):**
```bash
cd blockchain
npm install
cd ..
```

### 4️⃣ Run the Project

#### **All Services at Once (Recommended)**

From the project root, run:
```bash
npm run dev
```

This will automatically start:
- ✅ **Frontend** on http://localhost:3000
- ✅ **Backend API** on http://localhost:5000
- ✅ **Blockchain (Hardhat RPC)** on http://localhost:8545
- ✅ **ML Service (Flask)** on http://localhost:5002

#### **Run Services Individually**

If you prefer to run them separately in different terminals:

**Terminal 1 - Blockchain & Backend:**
```bash
npx cross-env npm run start:backend
```

**Terminal 2 - Frontend:**
```bash
npm start
```

**Terminal 3 - ML Service:**
```bash
cd ml
python app.py
```

## 🔐 Email Configuration (Optional)

To enable real email notifications, add your SMTP credentials to `.env`:

```bash
# For Gmail:
EMAIL_USER=your-email@gmail.com
EMAIL_PASS=your-app-password
# Get App Password: https://myaccount.google.com/apppasswords

# For other SMTP providers, update accordingly:
EMAIL_HOST=your-smtp-host
EMAIL_PORT=587
```

**Without email setup**, the system will log notifications to console during development.

## 📝 Environment Variables Explained

| Variable | Purpose | Default | Example |
|----------|---------|---------|---------|
| `PORT` | Backend server port | 5000 | 5000 |
| `REACT_APP_API_BASE_URL` | Frontend API endpoint | localhost:5000 | http://localhost:5000 |
| `BLOCKCHAIN_RPC_URL` | Hardhat RPC endpoint | 127.0.0.1:8545 | http://127.0.0.1:8545 |
| `WILL_AUTO_FUND_ETH` | Auto-fund amount for wills | 1 | 1 |
| `WILL_MIN_EXECUTION_FUND_ETH` | Min funds for execution | 1 | 1 |
| `ML_SERVICE_URL` | Certificate validation service | localhost:5002 | http://localhost:5002 |
| `EMAIL_USER` | SMTP email address | (empty) | your-email@gmail.com |
| `EMAIL_PASS` | SMTP password/app-password | (empty) | your-app-password |

## 🐛 Troubleshooting

### Port Already in Use
If a port is already in use, change it in `.env`:
```bash
# .env
PORT=5001              # Change backend port
REACT_APP_PORT=3001    # Change frontend port
ML_SERVICE_PORT=5003   # Change ML service port
```

### Email Not Sending
Ensure credentials are correct and SMTP port is accessible:
```bash
# Check if credentials work:
# EMAIL_USER=your-email@gmail.com
# EMAIL_PASS=your-app-password  (NOT your regular password)
```

### Blockchain Connection Error
Verify Hardhat is running:
```bash
# Should be running on port 8545
lsof -i :8545
```

### Python Dependencies Issue
```bash
cd ml
pip install --upgrade -r requirements.txt
```
